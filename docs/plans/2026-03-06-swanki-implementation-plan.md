# Swanki Web App Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a modern Anki-compatible flashcard web app with FSRS scheduling, multi-format import, and enhanced statistics.

**Architecture:** TanStack Start with API routes (`/api/*`) consumed by TanStack Query. SQLite database via Drizzle ORM. better-auth for authentication. ts-fsrs for spaced repetition scheduling.

**Tech Stack:** TanStack Start, React 19, bun:sqlite, Drizzle ORM, TanStack Query, shadcn/ui, Tailwind CSS v4, better-auth, ts-fsrs, Vitest

**Design Doc:** `docs/plans/2026-03-06-swanki-web-app-design.md`

### Task 4: Define Database Schema

**Files:**

- Create: `apps/web/src/db/schema.ts`

Define all application tables using Drizzle's `sqliteTable`:

**decks** — `id` (text PK), `userId`, `name`, `parentId` (nullable), `description`, `settings` (JSON with newCardsPerDay/maxReviewsPerDay), `createdAt`, `updatedAt`. Indexes on userId, parentId.

**noteTypes** — `id` (text PK), `userId`, `name`, `fields` (JSON array of {name, ordinal}), `css`, `createdAt`, `updatedAt`. Index on userId.

**cardTemplates** — `id` (text PK), `noteTypeId`, `name`, `ordinal`, `questionTemplate`, `answerTemplate`. Index on noteTypeId.

**notes** — `id` (text PK), `userId`, `noteTypeId`, `fields` (JSON Record<string,string>), `tags` (space-delimited text), `createdAt`, `updatedAt`. Indexes on userId, noteTypeId.

**cards** — `id` (text PK), `noteId`, `deckId`, `templateId`, `ordinal`, `due` (timestamp), `stability` (real), `difficulty` (real), `elapsedDays`, `scheduledDays`, `reps`, `lapses`, `state` (int: 0=new/1=learning/2=review/3=relearning), `lastReview`, `createdAt`, `updatedAt`. Indexes on noteId, deckId, due, state.

**reviewLogs** — `id` (text PK), `cardId`, `rating` (1-4), `state`, `due`, `stability`, `difficulty`, `elapsedDays`, `lastElapsedDays`, `scheduledDays`, `reviewedAt`, `timeTakenMs`. Indexes on cardId, reviewedAt.

**media** — `id` (text PK), `userId`, `filename`, `hash` (SHA-256), `mimeType`, `size`, `createdAt`. Indexes on userId, hash.

Auth tables (user, session, account, verification) will be generated by better-auth CLI in the next task.

See Task 14 for the full schema code.

**Step 1:** Create the schema file with all tables above.

**Step 2:** Commit after auth schema generation in next task.

### Task 7: Auth Middleware Helper

**Files:**

- Create: `apps/web/src/lib/auth-middleware.ts`

```typescript
// apps/web/src/lib/auth-middleware.ts
import { auth } from "./auth";

export async function getSession(request: Request) {
  return auth.api.getSession({ headers: request.headers });
}

export async function requireSession(request: Request) {
  const session = await getSession(request);
  if (!session) {
    throw new Response(JSON.stringify({ error: "Unauthorized" }), {
      status: 401,
      headers: { "Content-Type": "application/json" },
    });
  }
  return session;
}
```
